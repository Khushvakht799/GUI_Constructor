В ТЗ цель  "создать программу с графическим интерфейсом, которая создает Гуи интерфейс на другие программы, указанные пользователем. 
Которая способна анализировать каталог программы, устанавливать зависимопсти, рефакторить. 
И создать готовый под-ключ Гуи интерфейс для каталога, который пользователь указал".


Полный функционал GUI_Constructor (глобальная архитектура)
1. Меню верхнее (Windows‑like / IDE style)

Файл → Открыть проект, Сохранить, Выход

Анализ → Сканировать каталог, показать структуру, зависимости, entry points

Рефакторинг → Выбор модулей, запуск запланированных изменений, применение автоматических исправлений

Тестирование → Выбор модулей, запуск unit/functional тестов, отображение результата

Knowledge Base → Просмотр, редактирование, добавление новых ошибок и решений

Настройки → Пути к проекту, KB, конфигурации, режимы запуска (debug/verbose/auto-fix)

Глоссарий глаголов → просмотр и редактирование словаря команд для конструкторов

2. Динамическое подокно “Информация”

Отображение текущего выбранного модуля и действия

Динамическое обновление при выборе меню, дерева проекта или действий

Логи выполнения всех операций, включая анализ, рефакторинг, тестирование

3. Дерево проекта / файлов

Отображение структуры каталога

Синхронизация с фактическими файлами после изменений

Drag‑and‑drop модулей для упрощённого управления

Возможность выбора модуля для рефакторинга или тестирования через дерево

4. Knowledge Base (KB)

Загрузка/сохранение knowledge.json

Автоматическое применение решений к ошибкам

Самообучение: добавление новых записей при неизвестных ошибках

Возможность интерактивного редактирования через GUI

5. Когнитивные функции и самообучение

Отслеживание успешности применённых исправлений

Рефлексивное расширение базы знаний: автоматическое добавление новых знаний

Использование словаря глаголов для связывания действий с командами и модулями

Автоматическая генерация рекомендаций по рефакторингу и исправлению ошибок

6. Рефакторинг / Авто‑рефакторинг

Планирование изменений для выбранных модулей

Проверка зависимостей и предупреждение о циклах

Тестирование после рефакторинга

Взаимодействие с KB для исправлений

Возможность отката изменений

7. Тестирование

Запуск модульных и интеграционных тестов

Отображение результатов в панели “Информация”

Автоматическое логирование ошибок и обновление KB

8. Автогенерация GUI под ключ для выбранного проекта

Создание GUI для любого Python проекта по выбранной структуре

Использование шаблонов (Python/Tkinter, Web, CLI, Data Processor)

Настройка кнопок, полей и панели логов через словарь глаголов и KB

Поддержка запуска подмодулей через меню

9. Логи / Подокно вывода

Очередь логов для потоков, чтобы GUI не зависал

Отображение прогресса операций и ошибок

История действий с возможностью копирования в KB

10. Обзор / Запуск файлов

Диалог выбора файлов / папок

Запуск выбранного скрипта в отдельном потоке

Интеграция с KB для анализа ошибок и предложений исправлений

11. Словарь глаголов / глоссарий команд

Том 1: глаголы действий → действия → исполнение

Используется для генерации кнопок, меню и команд внутри GUI

Возможность расширения через GUI

12. Настройки / Конфигурация

Пути к проекту, KB, конфигурационным файлам

Режимы запуска модулей (debug/verbose/auto-fix)

Сохранение состояния GUI между сессиями

13. Стиль и UX

Windows‑like, IDE‑style (панели, меню, подокна)

Возможность расширения меню при добавлении новых модулей

Интуитивная навигация для пользователя

14. Дополнительно / продвинутые функции

Поддержка скриншотов и документации (PrintScreen)

Автоматическое создание под‑GUI для модулей на основе анализа

Динамическое связывание действий с глаголами и KB
